ESP32-CAM UDP Control Tester

⚠️ Browser Limitation: Web browsers cannot directly send UDP packets due to security restrictions. UDP testing must be done using external tools.

Testing UDP Control

Your ESP32-CAM is now listening for UDP commands on Port 8888.

Method 1: Python Script (Recommended)

Use the included test_udp_control.py script:

python test_udp_control.py YOUR_ESP32_IP

Method 2: Command Line (Linux/Mac)

echo "FWD" | nc -u -w1 YOUR_ESP32_IP 8888

Method 3: Command Line (Windows PowerShell)

$udpClient = New-Object System.Net.Sockets.UdpClient
$bytes = [System.Text.Encoding]::ASCII.GetBytes("FWD")
$udpClient.Send($bytes, $bytes.Length, "YOUR_ESP32_IP", 8888)
$udpClient.Close()

Method 4: Mobile Apps

  • Android: "UDP Sender" or "Network Analyzer"
  • iOS: "UDP Test Tool" or "Network Utility"

Available Commands:

  • FWD - Move forward
  • BWD - Move backward
  • LEFT - Turn left
  • RIGHT - Turn right
  • STOP - Stop motors
  • LED_ON - Turn LED on
  • LED_OFF - Turn LED off
  • SPEED+ - Increase speed
  • SPEED- - Decrease speed
  • REBOOT - Reboot ESP32

Why UDP?

UDP (User Datagram Protocol) is ideal for robot control because:

  • ✅ Lower latency - No connection handshake
  • ✅ Works better on congested networks
  • ✅ Lighter weight than TCP
  • ✅ No connection state to maintain
  • ✅ Can penetrate some firewall/NAT scenarios better

When to use UDP: When you experience lag or connection drops with the web interface (HTTP/TCP)
